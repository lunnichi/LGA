# Lattice Gas Automata (LGA) — Симуляция двумерного потока жидкости

## О проекте

Данный проект представляет собой реализацию клеточного газового автомата (Lattice Gas Automata, LGA) на языке C для моделирования двумерного течения жидкости (или газа) на гексагональной решётке.

Модель основана на классическом подходе FHP (Frisch–Hasslacher–Pomeau), в котором жидкость представляется как совокупность дискретных частиц, движущихся по фиксированным направлениям. Физика симуляции (столкновения) базируется на строгом соблюдении законов сохранения массы и импульса.

**Основная цель проекта:**
- Смоделировать поток жидкости в канале (аэродинамической трубе);
- Реализовать обтекание твердого препятствия (цилиндра);
- Продемонстрировать эффект образования ударной волны и аэродинамической тени (вихревого следа);
- Настроить визуализацию распределения плотности газа.

Проект разработан в рамках дисциплины:
> «Императивное программирование»

# Теоретическая основа

## Что такое Lattice Gas Automata

Lattice Gas Automata (LGA) — это дискретная вычислительная модель жидкости, в которой:
- Пространство разбито на дискретную решётку;
- Частицы находятся строго в узлах решётки;
- Частицы могут двигаться только по фиксированным векторам;
- Взаимодействие происходит через локальные микро-столкновения, макроскопический результат которых подчиняется уравнениям гидродинамики (Навье–Стокса).

---

# Используемая модель

В проекте используется модель FHP:
- Гексагональная решётка;
- 6 направлений движения частиц + 1 покоящаяся (rest) частица;
- Классическая схема обновления: **Collision** (столкновение) + **Streaming** (перенос).

Каждая клетка (узел решетки) содержит битовую маску наличия частиц и флаг препятствия:

```c
typedef struct 
{
    uint8_t state;       // Битовая маска частиц (0-6 биты).
    bool is_obstacle;    // Является ли клетка твердым телом.
} Cell;
```

# Структура состояний

Каждая частица кодируется отдельным битом:

| Бит | Направление |
|---|---|
| 0 | Восток |
| 1 | Северо-восток |
| 2 | Северо-запад |
| 3 | Запад |
| 4 | Юго-запад |
| 5 | Юго-восток |
| 6 | Rest-частица |

Всего существует 128 возможных состояний клетки.


# Основные возможности проекта

## Динамическая физика (Collision)
- Автоматическая генерация таблицы столкновений (lookup table) на лету;
- Программа перебирает все 128 состояний, вычисляет их массу и векторный импульс, и подбирает только физически допустимые исходы для симуляции вязкости.

## Точная геометрия (Streaming)
- Идеальная обработка сдвигов для гексагональной Offset-решётки (учет четных и нечетных строк).

## Граничные условия
- Bounce-Back (Обратный отскок): При ударе о препятствие или верхнюю/нижнюю стенку трубы вектор частицы разворачивается на 180 градусов (условие прилипания).
- Inlet (Вход): На левой границе постоянно генерируется монолитная 100% плотная стена частиц (эффект нагнетающего поршня/насоса).
- Outlet (Выход): Свободный выход газа за правую границу экрана.

## Препятствие
- Цилиндрический объект, сдвинутый в левую часть трубы (w / 3.0), чтобы оставить максимум места для наблюдения аэродинамического следа.

## Визуализация
- Сохранение покадровых графических файлов в формате PPM;
- Цветовое кодирование: черный (вакуум), сине-голубой градиент (газ, зависящий от плотности), серый (твердое препятствие).

# Описание файлов

## main.c
Главный управляющий модуль программы:
- настройка параметров симуляции (разрешение, время, плотность);
- динамическое выделение памяти для двумерных массивов;
- запуск главного цикла симуляции;
- вызов функций визуализации и смены буферов.

## physics.c
Физическое ядро симуляции:
- `init_grid`: инициализация сетки и установка цилиндрического препятствия;
- `init_collision_table`: динамическая генерация физически допустимых исходов столкновений FHP;
- `collide`: фаза микро-столкновений;
- `stream`: гексагональный перенос, граничные условия и генерация плотного Inlet-потока.

## io.c
Подсистема ввода-вывода и аналитики:
- `save_ppm`: отрисовка и сохранение цветовых кадров симуляции;
- `save_bin` / `load_bin`: работа с бинарными дампами состояния;
- `calc_avg_density`: расчёт макроскопической плотности.

## lga.h
Заголовочный файл проекта:
- определение структуры `Cell` (битовая маска направлений и флаг твердого тела);
- прототипы всех функций симуляции.

## test.c
Модульные и интеграционные тесты:
- проверка закона сохранения массы в замкнутой системе;
- тестирование стабильности решетки.


# Алгоритм работы

Каждая итерация симуляции строго разделена на два этапа:

## 1. Collision (Столкновение)
Частицы внутри клетки сталкиваются и изменяют направления согласно законам сохранения импульса и массы.
Collision реализован через lookup-table:

```c
uint8_t col_table[2][128];
```

Где индекс 0 — правила для обычной среды (газа), а 1 — для препятствий.
Это обеспечивает сложное поведение газа при O(1) доступе к исходу.

## 2. Streaming (Перенос)
После collision частицы перемещаются:
- в соседние клетки;
- по своим векторам;
- с учётом offset-layout гексагональной решетки.

---

# Offset-layout
Гексагональная решётка хранится в прямоугольном двумерном массиве.

Чётные и нечётные строки смещены относительно друг друга по оси X:

```text
even row:   o o o o
odd row:     o o o o
```

---

# Граничные условия

## Верхняя и нижняя стенки

Используется bounce-back:
- частица бьется о стенку;
- направление инвертируется (d + 3) % 6.

## Левая граница (Inlet)

Создаётся постоянная монолитная волна:
- на каждом шаге левый край принудительно заполняется частицами, летящими вправо (биты 0, 1, 5).

## Правая граница (Outlet)

Используется outflow:
- частицы покидают систему (игнорируются при выходе за пределы массива).

---

# Препятствие

Внутри канала размещён цилиндр.
Чтобы оставить больше места для вихревого следа, цилиндр смещен влево: cx = w / 3.0.

Если клетка принадлежит цилиндру:
- она помечается как is_obstacle = true;
- летящие в нее частицы отражаются обратно по вектору (Bounce-back) прямо в фазе stream.

